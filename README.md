# Common Dev Prompt

Claude Code 開發工作流的通用 Prompt 集，幫助團隊快速建立 AI 協作規範。

## 這是什麼？

一系列可複用的 Prompt，用來教 Claude Code「怎麼幫你做事」。
直接複製 Prompt 到任何專案中貼上執行，AI 會自動掃描專案結構並產生：

- **指引文件**（CLAUDE.md）— 讓 AI 理解你的專案架構與開發慣例
- **Agent Skill**（.claude/skills/）— 讓 AI 學會專案特有的協作技能
- **Spec-before-Code 工作流** — 強制 AI 先寫規格、確認後才寫程式碼

## 如何使用

1. 到目標專案根目錄開啟 Claude Code
2. 選擇適合的 Prompt，複製貼上
3. AI 自動掃描專案結構並產生所有檔案

不需要手動填入資料夾名稱或技術棧，AI 會自動偵測。

## 設計理念：原則導向，不綁死流程

這些 Prompt 的設計遵循一個核心理念：**像好的管理者一樣指導 AI — 定方向、設護欄，但不規定每一步怎麼走。**

AI 模型持續升級，每一版都有新的能力與認知。如果 Prompt 寫得太死（固定步驟、硬編數字、強制格式），反而會限制 AI 的發揮，讓它無法運用自身最新的判斷力。

### 原則

| 做法 | 說明 |
| --- | --- |
| **定義目標與約束，不定義步驟** | 告訴 AI「要達成什麼」和「不能踩的線」，具體怎麼做讓它依情境判斷 |
| **設底線，不設天花板** | 護欄防止出錯，但不限制 AI 用更好的方式完成任務 |
| **描述原則，不描述流程** | 流程會過時，原則不會；讓 AI 根據當下情境自己規劃最佳路徑 |
| **用彈性描述取代硬數字** | 「有一定複雜度的改動」比「超過 3 行的改動」更能適應不同場景 |

### 實務上的意思

- Skill 的步驟是**參考**，不是強制 SOP — AI 可以依情境調整順序或省略不必要的步驟
- 模板的欄位分**必要**與**選用** — AI 依改動規模決定文件深度，小改動不需要填滿每個欄位
- 工作流強調的是**意圖**（先想清楚再動手）而不是**形式**（一定要產出某種格式的文件）

> 寫 Prompt 跟帶團隊一樣：微管理出不了好成果，給方向和信任才行。

## 適用對象

任何使用 Claude Code 的開發團隊，無論是單一專案或多專案結構皆適用。
