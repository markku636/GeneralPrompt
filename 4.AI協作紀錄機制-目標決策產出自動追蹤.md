# 請幫我建立 AI 協作紀錄機制

> 使用方式：到目標專案根目錄開啟 Claude Code，複製「一鍵產生」區塊貼上即可。
> Node.js 需在 PATH 中可用（`node --version` 確認）。

---

每次與 AI 的協作過程都值得被記錄，包含目標是什麼、做了哪些決策、為什麼不採納某個建議，以及最終產出了什麼。這個機制透過 Hook 與規則，讓記錄自動發生，不需要使用者額外指示。

- **目標確認**：Session 開始時，AI 詢問並確認本次目標，存入協作紀錄
- **對話記錄**：AI 自動將關鍵問答與決策摘要追加至紀錄
- **不採納追蹤**：目標未確認時 Hook 攔截工具呼叫，自動寫入攔截紀錄；AI 亦自行記錄被拒絕的建議
- **產出摘要**：Session 結束前，AI 自動補齊程式碼變更、設計思路、測試案例等項目

## 新增目錄結構

```text
.claude/
├── settings.json              ← 新增 SessionStart / PreToolUse / SessionEnd hook 設定
└── hooks/
    ├── session-start-collab.mjs   ← SessionStart：建立紀錄檔、儲存路徑、提示確認目標
    ├── pre-tool-use-collab.mjs    ← PreToolUse：目標未確認時攔截 Edit/Write 並寫入紀錄
    └── session-end-collab.mjs     ← SessionEnd：提示 AI 補齊產出摘要

.claude/rules/
└── ai-collab-log.md           ← AI 自主行為規則（目標確認 / 對話記錄 / 決策追蹤 / 產出摘要）

docs/collab-logs/
├── README.md
├── _collab-template.md        ← 協作紀錄範本
└── {YYYYMMDD}-{NNN}-session.collab.md  ← 每次 Session 自動產生
```

## 觸發時機對照表

| 時機 | 機制 | 執行動作 |
| --- | --- | --- |
| Session 開始 | Hook（SessionStart） | 建立新 `.collab.md` 路徑，寫入 `.claude/.active-collab-log` |
| Session 開始 | 規則（AI 自主） | 詢問目標，確認後建檔並填入「目標確認」區塊 |
| AI 嘗試 Edit/Write，目標尚未確認 | Hook（PreToolUse） | 攔截工具呼叫，自動寫入「工具攔截紀錄」 |
| 重要決策確定後 | 規則（AI 自主） | 追加一筆至「關鍵對話摘要」表格 |
| 用戶拒絕 AI 建議 | 規則（AI 自主） | 在「決策記錄」標記 ❌ 並記錄理由 |
| Session 結束 | Hook（SessionEnd） | 檢查產出摘要是否為空，空則提示 AI 補填 |
| 任務完成 / Session 結束 | 規則（AI 自主） | 更新「產出摘要」區塊 |

## 一鍵產生

````text
請幫我在這個專案建立「AI 協作紀錄系統」，自動追蹤每次 AI 對話的目標、決策與產出。

依序建立以下檔案，若 `.claude/settings.json` 已存在則合併 hooks 設定（不覆蓋現有項目）：

---

### 1. .claude/settings.json

若已存在，在 `SessionStart`、`PreToolUse`、`SessionEnd` 陣列中各追加一個 hook 項目；若不存在則建立：

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          { "type": "command", "command": "node .claude/hooks/session-start-collab.mjs" }
        ]
      }
    ],
    "PreToolUse": [
      {
        "hooks": [
          { "type": "command", "command": "node .claude/hooks/pre-tool-use-collab.mjs" }
        ]
      }
    ],
    "SessionEnd": [
      {
        "hooks": [
          { "type": "command", "command": "node .claude/hooks/session-end-collab.mjs" }
        ]
      }
    ]
  }
}
```

### 2. .claude/hooks/session-start-collab.mjs

```javascript
#!/usr/bin/env node
import { readdir, readFile, writeFile, mkdir } from 'fs/promises';

const COLLAB_DIR = 'docs/collab-logs';
const STATE_FILE = '.claude/.active-collab-log';

async function nextSeq(dateStr) {
  try {
    const files = await readdir(COLLAB_DIR);
    const nums = files
      .filter(f => f.startsWith(dateStr) && f.endsWith('.collab.md'))
      .map(f => parseInt(f.match(/-(\d{3})-/)?.[1] ?? '0'));
    return nums.length ? String(Math.max(...nums) + 1).padStart(3, '0') : '001';
  } catch { return '001'; }
}

const now = new Date();
const d = `${now.getFullYear()}${String(now.getMonth()+1).padStart(2,'0')}${String(now.getDate()).padStart(2,'0')}`;

await mkdir(COLLAB_DIR, { recursive: true });
await mkdir('.claude', { recursive: true });

const seq = await nextSeq(d);
const logPath = `${COLLAB_DIR}/${d}-${seq}-session.collab.md`;
await writeFile(STATE_FILE, logPath, 'utf-8');

let recentCtx = '';
try {
  const files = (await readdir(COLLAB_DIR))
    .filter(f => f.endsWith('.collab.md') && !f.startsWith('_'))
    .sort().slice(-3);
  for (const f of files) {
    try {
      const c = await readFile(`${COLLAB_DIR}/${f}`, 'utf-8');
      const m = c.match(/\*\*目標\*\*：(.+)/);
      if (m?.[1]?.trim()) recentCtx += `  ${f}: ${m[1].trim()}\n`;
    } catch {}
  }
} catch {}

console.log(`
══════════════════════════════════════════════
  AI 協作紀錄系統 ── Session 開始
══════════════════════════════════════════════
新協作紀錄路徑：${logPath}
${recentCtx ? `\n近期紀錄：\n${recentCtx}` : ''}
⚡ [ai-collab-log] 請在首次回應前：
   1. 詢問用戶本次對話目標
   2. 確認後，建立 ${logPath} 並填入「目標確認」
══════════════════════════════════════════════
`);
```

### 3. .claude/hooks/pre-tool-use-collab.mjs

```javascript
#!/usr/bin/env node
import { readFile, writeFile, mkdir } from 'fs/promises';
import { existsSync } from 'fs';
import { dirname } from 'path';

process.stdin.setEncoding('utf8');
let raw = '';
for await (const chunk of process.stdin) raw += chunk;
if (!raw.trim()) process.exit(0);

let payload;
try { payload = JSON.parse(raw); } catch { process.exit(0); }

const tool = payload.tool_name ?? '';
if (!['Edit', 'Write', 'MultiEdit'].includes(tool)) process.exit(0);

const target = payload.tool_input?.file_path ?? payload.tool_input?.path ?? '';
if (target.includes('collab-logs') || target.includes('.active-collab-log')) process.exit(0);

let logPath;
try { logPath = (await readFile('.claude/.active-collab-log', 'utf-8')).trim(); }
catch { process.exit(0); }

let confirmed = false;
if (existsSync(logPath)) {
  try {
    const c = await readFile(logPath, 'utf-8');
    confirmed = /\*\*目標\*\*：\S/.test(c);
  } catch {}
}
if (confirmed) process.exit(0);

// Block: record intercept and exit non-zero
const ts = new Date().toISOString().slice(0, 16).replace('T', ' ');
const row = `| ${ts} | ${tool} | ${target || '(未知)'} | 尚未確認對話目標 |`;

try {
  if (!existsSync(logPath)) {
    await mkdir(dirname(logPath), { recursive: true });
    const blank = [
      `# AI 協作紀錄：(未命名)`,
      `> 建立：${ts}`,
      ``,
      `## 目標確認`,
      `- **目標**：`,
      `- **確認時間**：`,
      ``,
      `## 關鍵對話摘要`,
      `| # | 提示重點 | AI 回應摘要 |`,
      `|---|---------|------------|`,
      ``,
      `## 決策記錄`,
      `| # | 決策內容 | 採納 | 理由 |`,
      `|---|---------|:----:|------|`,
      ``,
      `### 工具攔截紀錄（Hook 自動寫入）`,
      `| 時間 | 工具 | 目標檔案 | 攔截原因 |`,
      `|------|------|---------|---------|`,
      ``,
      `## 產出摘要`,
      `- **程式碼變更**：`,
      `- **設計思路**：`,
      `- **測試案例**：`,
      `- **其他產出**：`,
    ].join('\n');
    await writeFile(logPath, blank, 'utf-8');
  }
  let c = await readFile(logPath, 'utf-8');
  c = c.replace('\n## 產出摘要', `\n${row}\n\n## 產出摘要`);
  await writeFile(logPath, c, 'utf-8');
} catch {}

process.stderr.write(`[AI 協作紀錄] 攔截：尚未確認對話目標\n請先確認目標並填入 ${logPath}，再繼續操作。\n`);
process.exit(2);
```

### 4. .claude/hooks/session-end-collab.mjs

```javascript
#!/usr/bin/env node
import { readFile, unlink } from 'fs/promises';

let logPath;
try { logPath = (await readFile('.claude/.active-collab-log', 'utf-8')).trim(); }
catch { process.exit(0); }

let summaryEmpty = true;
try {
  const c = await readFile(logPath, 'utf-8');
  const m = c.match(/## 產出摘要\n([\s\S]*)$/);
  if (m) summaryEmpty = !m[1].split('\n').some(l => /^- \*\*[^*]+\*\*：\S/.test(l));
} catch {}

try { await unlink('.claude/.active-collab-log'); } catch {}

if (summaryEmpty) {
  console.log(`
══════════════════════════════════════════════
  AI 協作紀錄系統 ── Session 結束
══════════════════════════════════════════════
⚡ [ai-collab-log] 產出摘要尚未填寫
請在結束前，更新 ${logPath} 的「產出摘要」：
  - 程式碼變更
  - 設計思路
  - 測試案例
  - 其他產出
══════════════════════════════════════════════
`);
} else {
  console.log(`\n✅ AI 協作紀錄完整：${logPath}\n`);
}
```

### 5. .claude/rules/ai-collab-log.md

```markdown
# AI 協作紀錄規則

本規則由 session-start-collab.mjs 與 session-end-collab.mjs 觸發，AI 自主執行，不需用戶額外指示。

## 規則一：目標確認（SessionStart 後）

SessionStart hook 輸出「AI 協作紀錄系統」訊息後，在首次回應前：

1. 詢問：「本次對話想解決什麼問題？」
2. 用戶確認後，建立 hook 指定的 `.collab.md` 檔案
3. 填入「目標確認」區塊：
   - **目標**：用戶確認的目標（一句話）
   - **確認時間**：當前時間（YYYY-MM-DD HH:mm）

## 規則二：關鍵對話記錄（對話進行中）

每完成重要決策或方向確定後，在「關鍵對話摘要」追加一筆：

| # | 提示重點 | AI 回應摘要 |
|---|---------|------------|
| n | 用戶核心問題 | 本次決策核心 |

## 規則三：不採納記錄

當以下情況發生，立即在「決策記錄」追加 ❌ 記錄：
- 用戶說「不要」、「改成」、「換個方式」、「取消」等
- 工具呼叫被用戶 Deny
- 你提出的方案被用戶否定

格式：`| n | 被否定的建議 | ❌ | 用戶給出的理由（若有）|`

採納時標記 ✅。

## 規則四：產出摘要（SessionEnd 後或任務完成時）

SessionEnd hook 提示或任務明確完成時，更新「產出摘要」：
- **程式碼變更**：主要修改的檔案與變更說明
- **設計思路**：本次採用的方案與核心原因
- **測試案例**：驗證方式（若有）
- **其他產出**：文件、設定、腳本等

## 注意事項

- 協作紀錄路徑由 `.claude/.active-collab-log` 指定
- 不採納記錄以「可追溯性」為優先，不帶評判語氣
- 若無進行中的協作紀錄（無 `.active-collab-log`），本規則靜默跳過
```

### 7. docs/collab-logs/README.md

```markdown
# AI 協作紀錄

每次 Claude Code Session 自動產生一份協作紀錄，記錄目標確認、關鍵決策、不採納理由與產出摘要。

## 檔案命名
`{YYYYMMDD}-{NNN}-session.collab.md`

- `YYYYMMDD`：建立日期
- `NNN`：當日流水號（001 起）

## 欄位說明

| 欄位 | 填寫時機 | 填寫者 |
| --- | --- | --- |
| 目標確認 | Session 開始，用戶確認目標後 | AI |
| 關鍵對話摘要 | 每次重要決策確定後 | AI |
| 決策記錄 | 採納或不採納建議時 | AI |
| 工具攔截紀錄 | 目標未確認前嘗試 Edit/Write 時 | Hook 自動 |
| 產出摘要 | Session 結束前或任務完成時 | AI |
```
````
